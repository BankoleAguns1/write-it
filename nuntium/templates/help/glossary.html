Glossary

This is a glossary of terms for WriteIt.

WriteIt is a zero-config factory for building sites that empower citizens to send emails to representatives and politicians.

How does WriteIt work?

Definitions

  • description
  • moderation
  • name
  • rate limit
  • testing mode
description

The description is shown publicly, and should make it clear who the recipients are — that is, who can be written to using this site. Use it to explain who these people are (for example, "The elected members of Exampleville City Council, who oversee the budget and municipal resources of Exampleville, Examplecountry.").

moderation

If moderation is enabled, the site will not send messages until the site owner has approved them.

name

The name of the WriteIt instance is public: anyone can see it. It's best to use names which only contain alphanumberic characters and hyphens. Don't use spaces in the name, because it may appear as the subdomain name in the URL.

rate limit

The number of messages a single author can send in one day is controlled by the rate limit. If this is zero, then no limit is applied. When the limit is exceeded, the user sees a message telling them to try again tomorrow.

testing mode

A WriteIt site in testing mode will not send messages to the email addresses in its database — instead it will send them to the site's owner. Also, WriteIt doesn't publicise testing sites: whenever WriteIt displays a list of sites, testing sites are not shown (unless you're logged in as the sites' owner).
